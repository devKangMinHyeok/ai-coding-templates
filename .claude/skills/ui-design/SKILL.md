# /ui-design — 단순하지만 좋아 보이게

비개발자가 만들어도 "프로 같다"는 인상을 주는 UI 규칙. Linear·Vercel·Stripe의 원칙을 단순화했다.

## 1. 정제된 미니멀리즘
- 한 화면에서 **시선이 갈 곳은 최대 3곳.** 나머지는 조용하게.
- 장식(아이콘·보더·배지)은 **정보를 줄 때만.** 그라디언트 남발 금지.
- 여백은 빈 공간이 아니라 **묶어주는 도구**다.

## 2. 시각적 계층 — 색보다 크기·굵기·간격 먼저
- 1순위(핵심): 진한 색 + `font-semibold`
- 2순위(보조): 옅은 색(`text-zinc-500`) + 보통 굵기
- 3순위(메타): 더 작게(`text-xs`)
- **색으로 구분하기 전에 크기·굵기로 먼저 구분.**

## 3. 일관된 간격
- Tailwind 간격 스케일을 **정해서 반복** 사용: 예) `gap-2 / gap-4 / gap-6`, `p-4 / p-6`.
- 아무 값이나 쓰지 말고 몇 개로 통일하면 자동으로 정돈돼 보인다.

## 4. 상태를 반드시 처리 (가장 자주 빠뜨림)
모든 데이터 화면은 네 가지 상태를 갖는다:
- **빈 상태**: "아직 없어요 + 무엇을 하면 되는지" 안내
- **로딩 상태**: 스켈레톤/스피너
- **에러 상태**: 무엇이 잘못됐는지 + 어떻게 복구하는지
- **정상 상태**: 데이터 표시
빈/로딩/에러를 빼먹지 않는다.

## 5. 모든 클릭 요소엔 피드백
- 버튼·링크·카드에 **hover/focus 상태** 필수(`hover:`, `focus-visible:`).
- 누르면 즉시 반응(비활성화·로딩 표시).

## 6. 반응형 + 다크모드
- 모바일 폭부터 깨지지 않게(`sm:`, `md:`로 확장).
- 다크모드 색 쌍을 같이 준다(`text-zinc-900 dark:text-zinc-100`).

## 7. 접근성 기본
- 이미지엔 `alt`, 폼 요소엔 라벨. 색만으로 정보를 전달하지 않는다.
