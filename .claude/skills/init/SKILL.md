# /init — 내 프로젝트로 시작하기

이 템플릿을 받은 후 **처음 한 번** 실행한다. 샘플 데이터를 정리하고, 이 환경이 어떻게 동작하는지 알려준 뒤, 바로 프로젝트 방향을 잡는 인터뷰로 넘어간다.

## 전제
- `pnpm install`이 완료되어 있어야 한다. 안 됐으면 먼저 실행한다.
- 이미 `spec/mission.md`가 있으면 "이미 초기화된 프로젝트입니다. `/help`를 참고하세요." 라고 안내하고 멈춘다.

## 절차

### 1단계: 환영 & 환경 확인

다음을 출력한다:

```
👋 안녕하세요! 이 환경은 AI와 함께 웹 서비스를 만들기 위한 작업 공간입니다.

지금부터 3가지를 할 거예요:
  1. 샘플 데이터 정리 (30초)
  2. 이 환경이 뭔지 간단히 소개 (2분)
  3. 당신의 프로젝트 방향 잡기 (/interview)
```

환경 확인:
```bash
node -v          # Node.js 설치 확인
pnpm -v          # pnpm 설치 확인
git status       # git 초기화 확인
```

하나라도 실패하면 해당 설치 가이드를 안내하고 멈춘다.

### 2단계: 샘플 데이터 정리

다음 더미/샘플 파일을 삭제한다:

```bash
# 샘플 learnings (날짜가 2024-01-01이고 파일명에 sample이 포함된 것)
rm -f learnings/2024-01-01-sample-*.md

# 샘플 decisions (날짜가 2024-01-01이고 파일명에 sample이 포함된 것)
rm -f decisions/2024-01-01-sample-*.md
```

삭제 후:
```
✅ 샘플 데이터를 정리했습니다.
   - learnings/ → 비었음 (앞으로 작업하면서 자동으로 쌓입니다)
   - decisions/ → 비었음 (설계 결정이 자동으로 기록됩니다)
```

**삭제하지 않는 것:**
- `app/notes/` (노트 데모 앱) — 학습 참고용으로 남겨둔다. 자기 프로젝트를 만들면서 자연스럽게 교체된다.
- `server/note/` (노트 서버 코드) — Port & Adapter 패턴의 실제 예시로 남긴다.
- `components/notes-demo.tsx` — 컴포넌트 구조 참고용.
- `spec/TEMPLATE.md`, `spec/MISSION_TEMPLATE.md` — 양식은 유지.

### 3단계: 이 환경 소개

사용자에게 다음을 **대화체로 자연스럽게** 설명한다 (목록을 덤프하지 않는다):

#### 핵심 개념 (1분)

> **이 환경은 "가드레일"입니다.**
>
> 당신이 "이런 거 만들고 싶어"라고 말하면, 저는 그냥 코드를 짜는 게 아니라:
> - 먼저 **뭘 만들지 구체화**하고 (스펙)
> - **작게 나눠서** 구현하고
> - 매 단계마다 **자동으로 검증**하고
> - 배포 전에 **전문가 관점으로 점검**합니다.
>
> 이렇게 하면 "돌아가긴 하는데 엉망인 코드"가 아니라, **나중에도 고치고 키울 수 있는 코드**가 만들어집니다.

#### 5개 커맨드 (1분)

> **기억할 건 5개뿐입니다:**
>
> 1. **`/interview`** — "뭘 만들지?" 방향을 잡는다 (지금 할 거예요)
> 2. **`/spec`** — "구체적으로 어떤 기능?" 스펙을 만든다
> 3. **`/impl`** — 스펙대로 만든다 (자동 검증 포함)
> 4. **`/review`** — 배포 전 점검한다
> 5. **`/ship`** — 세상에 내보낸다 (push → 자동 배포)
>
> 나머지는 전부 뒤에서 자동으로 돌아갑니다. 모르겠으면 언제든 **`/help`**라고 하세요.

#### 자동으로 되는 것 (30초)

> **이런 건 신경 안 써도 됩니다:**
> - 코드 스타일 → ESLint가 자동으로 잡아줌
> - 커밋 전 검증 → Husky가 자동으로 막아줌
> - 배포 → push만 하면 Vercel이 자동으로
> - 배운 것 기록 → 작업하면서 자동으로 쌓임
> - 설계 결정 기록 → 판단할 때마다 자동으로 쌓임

### 4단계: /interview 로 이어가기

```
자, 이제 당신의 프로젝트를 시작할 시간입니다! 🎯
```

바로 `/interview` 스킬의 절차를 시작한다. (사용자에게 "/interview를 입력하세요"라고 하지 않고, 자연스럽게 첫 질문으로 넘어간다.)

> "한 문장으로 말해주세요 — 어떤 제품을 만들고 싶나요?"

## 주의사항

- `/init`은 **한 번만** 실행한다. `spec/mission.md`가 이미 있으면 중복 실행을 막는다.
- 환경 확인에서 문제가 있으면 초기화를 진행하지 않고, 문제 해결부터 안내한다.
- 소개는 **짧고 자연스럽게**. 매뉴얼을 읽어주는 느낌이 아니라 "같이 만들어봐요" 느낌으로.
- 노트 데모 앱은 남겨둔다 — 학생이 참고할 수 있는 유일한 실제 코드 예시.
