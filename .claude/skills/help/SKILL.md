# /help — 여기서 할 수 있는 것

이 저장소는 **가드레일이 미리 깔린 AI 코딩 하네스**다. 만들고 싶은 걸 말하면, 검증·구조·배포는 이 환경이 잡아준다. 패키지 매니저는 **pnpm**.

## 만들기 흐름
- **/interview** — 질답으로 "무엇을 만들지"를 좁혀 `spec/spec.md`를 만든다. (구현 안 함)
- **/impl** — 확정된 스펙을 작게 나눠 구현한다.
- **/review** — 배포 전 스스로 점검한다.
- **/ship** — 커밋·푸시해서 Vercel에 자동 배포한다.

## 코드를 잘 짜기 위한 가이드 (항상 적용)
- **architecture** — 어디에 코드를 둘지, 저장/서버 등 설계 판단.
- **server-architecture** — 서버 코드(저장·외부연동·비밀키)의 레이어 구조(포트&어댑터).
- **database** — 영속 저장이 필요할 때 Supabase(무료 플랜) 붙이기.
- **testing** — 순수 로직을 Vitest로 테스트(검증 게이트의 일부).
- **conventions** — const·순수함수·선언형 등 코드 규칙(ESLint로 강제).
- **ui-design** — 보기 좋고 일관된 화면.
- **/verify** — 타입·린트·테스트·빌드를 통과할 때까지 스스로 고친다.

## 표준 흐름
1. **/interview** → 2. **/impl** → 3. **/review** → 4. **/ship**
