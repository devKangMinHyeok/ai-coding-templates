# /add-review-profile — 커스텀 리뷰 프로필 만들기

프로젝트에 필요한 전문가 리뷰 프로필을 인터뷰 형태로 만든다. 만들어진 프로필은 `/review <이름>`으로 바로 사용할 수 있다.

## 전제

- `.claude/skills/review/profiles/` 폴더가 있어야 한다 (없으면 생성)

## 절차

사용자에게 차례로 질문한다:

### 1단계: 어떤 전문가인가?
> "어떤 전문가의 관점으로 리뷰하고 싶나요?"
> 예시: "보안 전문가", "접근성 전문가", "SEO 전문가", "데이터 분석가"

### 2단계: 이 전문가는 뭘 보나?
> "이 전문가가 코드를 볼 때 가장 먼저 확인하는 것은 뭔가요?"
> (3~5개 핵심 관심사를 뽑는다)

### 3단계: 어떤 파일을 봐야 하나?
> "주로 어떤 종류의 파일을 검토해야 할까요?"
> 예시: "서버 코드만", "모든 코드", "프론트엔드만", "설정 파일"
> (파일 패턴으로 변환: `server/**`, `**/*.tsx` 등)

### 4단계: 핵심 질문
> "이 전문가가 리뷰할 때 머릿속에 두는 한 가지 질문이 있다면?"
> 예시: "이 코드가 공격자에게 악용될 수 있는가?"

### 5단계: 체크리스트
2단계에서 뽑은 관심사를 기반으로 체크리스트를 생성한다.
사용자에게 확인:
> "이런 체크리스트로 리뷰하면 될까요? 추가하거나 빼고 싶은 항목이 있나요?"

### 6단계: 프로필 이름
> "이 프로필의 이름을 정해주세요. (영문 kebab-case, 예: security, seo, data-analyst)"

### 7단계: 파일 생성

`.claude/skills/review/profiles/{이름}.md`에 다음 구조로 저장:

```markdown
# {전문가 이름} Review Profile

{한 줄 설명}

## 페르소나

- {전문가 역할 설명}
- {리뷰 접근 방식}

**핵심 질문**: "{4단계에서 정한 질문}"

## 파일 필터

- {3단계에서 정한 패턴들}

## 참조 기준

1. {관련 스킬 문서 또는 spec 파일}

## 체크리스트

{5단계에서 확정된 체크리스트}

## 출력 포맷

(기존 프로필과 동일한 테이블 형식)

## 심각도 기준

| 심각도 | 기준 | 예시 |
|---|---|---|
| Critical | ... | ... |
| Warning | ... | ... |
| Suggestion | ... | ... |
```

### 8단계: 안내

> "✅ `{이름}` 리뷰 프로필이 만들어졌습니다!"
> "사용법: `/review {이름}`"
> "자동 감지에 추가하려면 `/review` SKILL.md의 파일 패턴 매핑 테이블에 추가하세요."

## 주의사항

- 기존 프로필과 이름이 겹치면 덮어쓰기 전에 확인을 묻는다
- 체크리스트는 3~7개 섹션, 각 섹션에 3~5개 항목이 적절하다
- 너무 넓은 범위(모든 것을 다 보는)는 피하고, 특정 관점에 집중하도록 유도한다
