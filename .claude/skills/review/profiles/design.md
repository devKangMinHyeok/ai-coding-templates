# Design Review Profile

UI를 **디자인 이론 + 프로젝트 디자인 철학** 기준으로 리뷰합니다.

## 페르소나

- Linear, Vercel, Stripe 수준의 디자인 감각을 가진 Product Designer
- "왜 이 간격인가, 왜 이 색상인가"를 시스템적으로 설명하는 리뷰어

**핵심 질문**: "이 UI가 깔끔하고 일관적이며, 모든 상태에서 잘 동작하는가?"

## 파일 필터

- `**/*.tsx` (React 컴포넌트·페이지)
- `app/**/*.css`

## 참조 기준

1. **`.claude/skills/ui-design/SKILL.md`** — UI 디자인 규칙

## 체크리스트

### A. 시각적 계층

- [ ] 글자 크기가 계층적인가? (제목 > 본문 > 보조)
- [ ] 라벨과 값이 시각적으로 구분되는가? (라벨은 연한 색, 값은 진한 색)
- [ ] 한 화면 안에서 텍스트 계층이 3단계 이내인가?
- [ ] 하드코딩 색상(`text-gray-500`) 대신 시맨틱 토큰(`text-muted-foreground`)을 사용하는가?

### B. 상태 처리

- [ ] **빈 상태**: 데이터가 없을 때 "아직 없어요" + 행동 유도가 있는가?
- [ ] **로딩 상태**: 데이터를 불러오는 동안 로딩 표시가 있는가?
- [ ] **에러 상태**: 뭔가 잘못됐을 때 사용자가 이해할 수 있는 메시지가 있는가?

### C. 간격 & 일관성

- [ ] 관련 요소 간 간격이 일관적인가?
- [ ] 여백이 시각적 그룹핑 역할을 하는가? (그룹 내 < 그룹 간)
- [ ] 카드/컨테이너 내부 패딩이 일관적인가?

### D. 인터랙션

- [ ] 클릭할 수 있는 요소에 hover 상태가 있는가?
- [ ] 트랜지션이 자연스러운가? (150-200ms)
- [ ] 포커스 상태(`focus-visible`)가 있는가?
- [ ] 아이콘 전용 버튼에 텍스트 라벨이나 tooltip이 있는가?

### E. 반응형 & 모바일

- [ ] 모바일 폭(375px)에서 레이아웃이 깨지지 않는가?
- [ ] 터치 타겟이 최소 44x44px인가?
- [ ] 텍스트가 모바일에서 잘리거나 넘치지 않는가?

### F. 다크 모드

- [ ] 하드코딩 색상(`bg-white`, `text-black`)이 없는가?
- [ ] 시맨틱 색상 토큰으로 다크 모드에서 자동 전환되는가?
- [ ] 그림자/보더가 다크 모드에서 적절한 대비를 유지하는가?

### G. 접근성

- [ ] 색상만으로 정보를 전달하지 않는가?
- [ ] 이미지에 alt 텍스트가 있는가?
- [ ] 폼 요소에 라벨이 연결되어 있는가?

## 심각도 기준

| 심각도 | 기준 | 예시 |
|---|---|---|
| Critical | 시각적 계층 역전, 접근성 위반 | 라벨이 값보다 굵음, focus ring 없음, 하드코딩 `bg-white` |
| Warning | 일관성 미흡, 간격 불일치, 상태 누락 | gap 혼용, 빈 상태 없음, 모바일 깨짐 |
| Suggestion | 더 나은 패턴 존재, 미세 개선 | hover 트랜지션 누락, tooltip 미적용 |
