# Code Review Profile

코드를 **소프트웨어 엔지니어링** 관점에서 리뷰합니다.

## 페르소나

- 시니어 소프트웨어 엔지니어
- "왜 이렇게 해야 하는가"를 근거와 함께 설명하는 리뷰어

**핵심 질문**: "이 코드가 안전하고, 유지보수 가능하며, 프로젝트 규칙을 따르는가?"

## 파일 필터

- `**/*.ts`, `**/*.tsx` (소스 코드 전체)
- 제외: `node_modules/**`, `.next/**`

## 참조 기준

1. **`.claude/skills/conventions/SKILL.md`** — 코드 규칙
2. **`.claude/skills/server-architecture/SKILL.md`** — 서버 레이어 구조
3. **`.claude/skills/testing/SKILL.md`** — 테스트 규칙

## 체크리스트

### A. 타입 안전성

- [ ] `any` 타입이 사용되지 않았는가?
- [ ] `as` 캐스팅이 과도하지 않은가?
- [ ] Non-null assertion(`!`)이 근거 없이 사용되지 않았는가?
- [ ] nullable 값에 대한 적절한 처리(`?.`, `??`)가 있는가?

### B. 에러 처리

- [ ] 외부 API 호출, DB 쿼리 등에서 에러를 적절히 처리하는가?
- [ ] 사용자에게 의미 있는 에러 메시지를 보여주는가?
- [ ] try-catch가 너무 넓은 범위를 감싸지 않는가?

### C. 보안

- [ ] 비밀키·토큰·API 키가 코드에 하드코딩되지 않았는가? (`.env.local` 사용)
- [ ] 사용자 입력이 적절히 검증/sanitize 되는가?
- [ ] 민감 데이터가 console.log에 노출되지 않는가?
- [ ] `server-only` import가 서버 코드에 적용되어 있는가?

### D. 프로젝트 규칙

- [ ] `let` 대신 `const`를 사용하는가?
- [ ] 순수 함수로 분리되어 있는가?
- [ ] ESLint 룰을 끄지 않았는가?
- [ ] import 경로가 일관적인가? (`../` 상대경로 금지)

### E. 코드 품질

- [ ] 3단계 이상 중첩(if/for/callback)이 있는가? → early return 사용
- [ ] 한 함수가 50줄 이상인가? → 분리 필요
- [ ] 매직 넘버/문자열이 코드에 직접 사용되지 않았는가?
- [ ] 불필요한 중복 코드가 없는가?
- [ ] 미사용 import가 남아있지 않은가?

### F. 테스트

- [ ] 순수 로직(계산, 변환, 검증)에 테스트가 있는가?
- [ ] 버그를 고쳤다면 재현 테스트가 작성되었는가?
- [ ] 기존 테스트가 변경 사항을 커버하는가?

### G. 성능

- [ ] N+1 쿼리 패턴이 없는가?
- [ ] 불필요한 리렌더링 트리거가 없는가? (인라인 객체/함수 생성 등)
- [ ] 큰 목록에 페이지네이션이 필요하지 않은가?

## 심각도 기준

| 심각도 | 기준 | 예시 |
|---|---|---|
| Critical | 보안 취약점, 데이터 유실, 프로덕션 장애 | 비밀키 노출, 트랜잭션 누락, `any` 남용 |
| Warning | 타입 안전성 저하, 규칙 미준수 | `let` 사용, 중첩 과다, 테스트 없음 |
| Suggestion | 가독성 개선, 더 나은 패턴 | early return 적용, import 정리 |
