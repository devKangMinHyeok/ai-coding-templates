# /review — 내보내기 전 스스로 점검

`/ship` 하기 직전에 돌린다. 빠르게 자기 결과물을 여러 관점으로 훑어 큰 구멍을 막는다.

## 체크리스트

**스펙**
- [ ] `spec/spec.md`의 "완성 기준"을 실제로 충족했나?
- [ ] 스펙에 없는 걸 욕심내서 넣지 않았나(스코프 크리프)?

**동작·상태**
- [ ] 빈 상태 / 로딩 / 에러 상태를 처리했나?
- [ ] 모바일 폭에서 깨지지 않나?

**안전**
- [ ] 비밀키(API 키 등)를 코드에 직접 박지 않았나? (`.env.local` 사용, 커밋 금지)
- [ ] 외부 입력을 그대로 신뢰하지 않았나?

**품질**
- [ ] `any`나 꺼둔 ESLint 룰이 없나?
- [ ] `pnpm verify`가 초록불인가?

**단순함**
- [ ] 더 단순하게 할 수 있는데 복잡하게 만든 곳은 없나?

→ 걸리는 게 있으면 고치고 다시. 다 통과하면 `/ship`.
