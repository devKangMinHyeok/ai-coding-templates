# spec 폴더

여기에 **프로젝트의 방향**과 **무엇을 만들지**가 들어갑니다.

- `MISSION_TEMPLATE.md` — 미션 양식 (수정하지 마세요)
- `TEMPLATE.md` — 스펙 양식 (수정하지 마세요)
- `mission.md` — `/interview` 스킬이 질답을 거쳐 만들어 줍니다 (프로젝트의 나침반)
- `spec.md` — `/spec` 스킬이 질답을 거쳐 만들어 줍니다 (만들 기능 정의)

## 순서
1. **`/interview`** → `mission.md` (프로젝트를 처음 시작할 때 한 번)
2. **`/spec`** → `spec.md` (기능을 만들 때마다)
3. **`/impl`** → 구현 시작
