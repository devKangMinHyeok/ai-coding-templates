# 저장소는 메모리 어댑터로 시작하고 나중에 Supabase로 교체

## 결정
Port & Adapter 패턴으로 저장소를 추상화하고, 기본은 메모리 어댑터(`repo.memory`)로 시작한다. 영속 저장이 필요해지면 Supabase 어댑터(`repo.supabase`)로 한 줄 교체한다.

## 이유
- 처음부터 DB를 붙이면 환경 설정이 복잡해져서 초기에 막힘
- 메모리로 시작하면 DB 없이도 기능 개발·테스트가 가능
- 어댑터만 바꾸면 되므로 비즈니스 로직은 수정할 필요 없음
- "아, 저장이 필요하구나"를 직접 느낀 후에 DB를 붙이는 자연스러운 흐름

## 고려한 대안
- **처음부터 Supabase**: 환경 설정 실패로 시간 낭비 위험
- **localStorage**: 서버 사이드에서 쓸 수 없음, SSR과 충돌
- **JSON 파일**: 배포 환경에서 파일 시스템 접근 불가 (Vercel은 읽기 전용)

## 되돌릴 조건
이 패턴 자체를 바꿀 이유는 거의 없음. 어댑터를 추가하는 것이지 기존 구조를 바꾸는 게 아님.
